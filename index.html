GearmanBundle

What is GearmanBundle?

GearmanBundle is a bundle for symfony2 intended provide a easy way support developers who need use job queues.

It uses annotations to define Jobs and Workers' options.

Requirements

To work with GearmanBundle needed, PHP 5.3, Gearman and Symfony2

Installation

You have two options to install it in Symfony2 framework: using vendors or by repository. The two options are based on the same basics, but one requires a little more configuration.

On the one hand we can set ourselves the repository as follows

                    ~/path/to/your/bundles[master]: git clone git://github.com/mmoreramerino/GearmanBundle.git

Moreover, you can include this repository in the list of vendors and reinstall again. You must edit your deps file in your project root

			~/path/to/your/project/deps
			[AsseticBundle]

			    git=http://github.com/mmoreramerino/GearmanBundle.git

			    target=/bundles/Mmoreramerino/GearmanBundle

When you have registered the bundle and you're in your project, you just need to instantiate your bundle and add a line in the autoload.

			~/path/to/your/project/app/AppKernel.php
			<?php


			$bundles = array(

			    new Symfony\Bundle\FrameworkBundle\FrameworkBundle(),

			    new Symfony\Bundle\TwigBundle\TwigBundle(),

			    new Symfony\Bundle\MonologBundle\MonologBundle(),

			    new Symfony\Bundle\SwiftmailerBundle\SwiftmailerBundle(),


			    ...


			    new Mmoreramerino\GearmanBundle\MmoreramerinoGearmanBundle(),

			);
                
			~/path/to/your/project/app/autoload.php
			<?php


			$loader->registerNamespaces(array(

    'Sensio'                  => __DIR__ . '/../vendor/bundles',

    'JMS'                     => __DIR__ . '/../vendor/bundles',

    'Doctrine\\Common'        => __DIR__ . '/../vendor/doctrine-common/lib',

    ...


    'Mmoreramerino'           => __DIR__ . '/../vendor/bundles',

			));
